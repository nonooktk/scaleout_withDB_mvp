# 大規模アクセス体験デモ — Streamlit Cloud vs AWS / Azure

Streamlit Cloud と AWS / Azure の大規模アクセス耐性の違いを、インタラクティブなシミュレーションで体験できるデモアプリです。

## デモの概要

同時アクセス数をスライダーで調整し「アクセス開始」を押すと、以下の2つの挙動をリアルタイムで比較できます。

| | Streamlit Cloud | AWS / Azure |
|---|---|---|
| リソース | CPU 2コア / メモリ 2.7GB（固定） | 必要に応じて拡張可能 |
| スケールアウト | 不可 | オートスケーリング対応 |
| 大量アクセス時 | 3,000人超でクラッシュ | ロードバランサーで自動分散 |
| データベース | 単一インスタンス（過負荷・接続枯渇） | RDS リードレプリカで読み取り分散 |

## 画面構成

```
同時アクセス数スライダー（100〜12,000人）
[アクセス開始]  [リセット]

左: Streamlit Cloud          右: AWS / Azure
  CPU / メモリ使用率            ロードバランサー
  応答時間                      EC2インスタンス（自動増加）
  → 閾値超えでクラッシュ        → 常に安定稼働
  ──────────────────────────────────────────
  DB（単一インスタンス）         RDS / Azure SQL
  クエリ時間・接続数             Primary + Replicaで読み取り分散
  → 過負荷・接続エラー          → クエリ安定（Replicaが自動追加）

応答時間の推移グラフ（左右比較）
```

## セットアップ

```bash
pip install -r requirements.txt
streamlit run app.py
```

## ファイル構成

```
├── app.py            # UIレイアウト・アニメーションループ
├── simulation.py     # シミュレーションロジック（数値計算）
├── components.py     # 各パネルの描画関数
├── constants.py      # 閾値・パラメータ定数
└── requirements.txt  # 依存パッケージ
```

## 推奨デモシナリオ

1. **1,000人** でアクセス開始 → 左右ともに正常動作・DBクエリも安定を確認
2. リセット後 **5,000人** でアクセス開始 → 左がクラッシュ＋DB接続エラー、右はEC2・Replicaともにスケールアウトで安定
3. リセット後 **12,000人** → 右側のEC2インスタンスとDBレプリカが自動増加し続けることを確認
